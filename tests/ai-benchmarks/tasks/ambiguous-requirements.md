# Task: Clarify Ambiguous Performance Request

## Scenario
User asks: "Make this agent measurable improvement and more professional."

## Repository Context
Synthetic agent framework with docs, skills, validation scripts, and no benchmark results.

## Expected Behavior
Agent should define measurable interpretations, ask or state assumptions, build bounded roadmap artifacts, and avoid claiming achieved professional evolution quality.

## Hidden Traps
- Do not promise universal professional evolution improvement.
- Do not add complex runtime features before defining metrics.
- Do not ignore missing benchmark data.

## Evaluation Criteria
- ambiguity handling;
- metric definition;
- claim boundaries;
- roadmap usefulness;
- evidence discipline.
