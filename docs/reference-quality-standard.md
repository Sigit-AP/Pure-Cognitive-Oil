# Reference Quality Standard

References must be useful, bounded, and auditable.

## Required Qualities

- Clear purpose.
- Task or decision boundary.
- Failure modes or red flags where relevant.
- No unsupported factual claims.
- No secrets.
- Link to validation or usage path when possible.

## Quality Gate

Reference quality review checks: duplicates, empty headings, unsafe claims, stale commands, missing evidence language, and files excluded from package output.
