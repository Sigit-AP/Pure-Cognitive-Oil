# Security Policy

## Supported Scope

This repository contains CLI scripts, agent skills, plugin surfaces, hooks, references, and validation tooling for local AI-agent workflows.

## Reporting

Report security issues through GitHub private vulnerability reporting when available. If unavailable, open a minimal public issue that says a private security report is needed and avoid exploit details.

## Do Not Include

- API keys, tokens, cookies, SSH keys, or private credentials.
- Working exploit payloads in public issues.
- Production secrets or user data.

## Maintainer Response

Maintainers should acknowledge, reproduce, patch, add validation where practical, and document residual risk.
